# হিসাব পত্র (Hisab Potro)

একটি সম্পূর্ণ ফার্মেসি বাকি হিসাব ব্যবস্থাপনা সিস্টেম যা PHP এবং MySQL এ তৈরি।

## বৈশিষ্ট্যসমূহ

### 🏪 মূল বৈশিষ্ট্য
- **ব্যবহারকারী ব্যবস্থাপনা**: ইমেল এবং Google OAuth দিয়ে লগইন
- **গ্রাহক ব্যবস্থাপনা**: গ্রাহকের তথ্য, যোগাযোগ, গ্রাম অনুযায়ী সার্চ
- **বাকি হিসাব**: বাকি দেওয়া/নেওয়া ট্র্যাকিং
- **পেমেন্ট ব্যবস্থাপনা**: বিভিন্ন পেমেন্ট মেথড, রশিদ জেনারেশন
- **রিয়েল-টাইম সিঙ্ক**: দুই ব্যবহারকারীর মধ্যে ডাটা সিঙ্ক

### 📊 রিপোর্টিং এবং অ্যানালিটিক্স
- **ড্যাশবোর্ড**: রিয়েল-টাইম স্ট্যাটিস্টিক্স এবং চার্ট
- **মাসিক রিপোর্ট**: মাস অনুযায়ী লেনদেনের হিসাব
- **খেলাপী তালিকা**: শীর্ষ খেলাপীদের তালিকা
- **মেয়াদোত্তীর্ণ বাকি**: মেয়াদ উত্তীর্ণ বাকির এলার্ট

### 🎨 UI/UX বৈশিষ্ট্য
- **রেসপন্সিভ ডিজাইন**: মোবাইল, ট্যাবলেট এবং ডেস্কটপ সাপোর্ট
- **বাংলা ফন্ট**: Hind Siliguri ফন্ট সাপোর্ট
- **আধুনিক UI**: Tailwind CSS দিয়ে তৈরি আকর্ষণীয় ইন্টারফেস
- **ইন্টারেক্টিভ চার্ট**: Chart.js দিয়ে ডাটা ভিজুয়ালাইজেশন

### 🔒 নিরাপত্তা বৈশিষ্ট্য
- **SQL ইনজেকশন প্রতিরোধ**: Prepared Statements ব্যবহার
- **XSS প্রতিরোধ**: Input sanitization এবং output escaping
- **CSRF প্রতিরোধ**: CSRF token validation
- **রোল ভিত্তিক অ্যাক্সেস**: Admin/User রোল ম্যানেজমেন্ট

## সিস্টেম প্রয়োজনীয়তা

### সার্ভার প্রয়োজনীয়তা
- **PHP**: 7.4 বা উচ্চতর
- **MySQL**: 5.7 বা উচ্চতর
- **Web Server**: Apache (পছন্দসই) বা Nginx
- **PHP Extensions**: PDO, PDO_MySQL, cURL, JSON, GD

### ক্লায়েন্ট প্রয়োজনীয়তা
- **ব্রাউজার**: Chrome 80+, Firefox 75+, Safari 13+, Edge 80+
- **ইন্টারনেট**: স্থিতিশীল ইন্টারনেট সংযোগ (রিয়েল-টাইম সিঙ্কের জন্য)

## ইনস্টলেশন

### ১. ডাউনলোড এবং আনজিপ
```bash
# প্রোজেক্ট ডাউনলোড করুন
git clone https://github.com/yourusername/hisab-potro.git

# অথবা ZIP ফাইল ডাউনলোড করে আনজিপ করুন
unzip hisab-potro.zip
cd hisab-potro
```

### ২. ডাটাবেস সেটআপ
```sql
# database.sql ফাইলটি আপনার MySQL সার্ভারে ইমপোর্ট করুন
mysql -u username -p database_name < database.sql
```

### ৩. কনফিগারেশন
ফাইলগুলো আপডেট করুন:

**config/database.php**
```php
define('DB_HOST', 'your_database_host');
define('DB_NAME', 'your_database_name');
define('DB_USER', 'your_database_user');
define('DB_PASS', 'your_database_password');
```

**config/google-config.php** (Google OAuth এর জন্য)
```php
define('GOOGLE_CLIENT_ID', 'your_google_client_id');
define('GOOGLE_CLIENT_SECRET', 'your_google_client_secret');
define('GOOGLE_REDIRECT_URI', 'https://yourdomain.com/callback.php');
```

### ৪. ফাইল পারমিশন
```bash
# লগ ফোল্ডার তৈরি করুন
mkdir logs

# পারমিশন সেট করুন
chmod 755 logs/
chmod 644 logs/*
chmod 644 config/*.php
```

### ৫. ওয়েব সার্ভারে আপলোড
ফাইলগুলো আপনার ওয়েব সার্ভারে আপলোড করুন এবং ওয়েব সাইটটি ভিজিট করুন।

## ব্যবহারের নির্দেশিকা

### প্রথম ব্যবহার
১. ওয়েবসাইটে ভিজিট করুন
২. "সাইন আপ" এ ক্লিক করুন
৩. আপনার তথ্য দিয়ে একাউন্ট তৈরি করুন অথবা Google দিয়ে লগইন করুন
৪. ড্যাশবোর্ড থেকে গ্রাহক, বাকি এবং পেমেন্ট ব্যবস্থাপনা শুরু করুন

### গ্রাহক ব্যবস্থাপনা
- **নতুন গ্রাহক যোগ করুন**: "নতুন গ্রাহক" বাটনে ক্লিক করে গ্রাহকের তথ্য দিন
- **গ্রাহক খুঁজুন**: নাম, ফোন বা গ্রাম দিয়ে সার্চ করুন
- **গ্রাহকের তথ্য দেখুন**: গ্রাহকের সম্পূর্ণ হিসাব দেখতে গ্রাহকের নামে ক্লিক করুন

### বাকি ব্যবস্থাপনা
- **বাকি যোগ করুন**: "নতুন বাকি" বাটনে ক্লিক করে বাকির তথ্য দিন
- **বাকির ধরন**: "বাকি দেয়া" বা "বাকি নেয়া" নির্বাচন করুন
- **মেয়াদ ঠিক করুন**: বাকি পরিশোধের মেয়াদ নির্ধারণ করুন

### পেমেন্ট ব্যবস্থাপনা
- **পেমেন্ট নিন**: "পেমেন্ট" বাটনে ক্লিক করে টাকা গ্রহণ করুন
- **পেমেন্ট মেথড**: ক্যাশ, ব্যাংক, মোবাইল ব্যাংকিং বা চেক নির্বাচন করুন
- **রশিদ প্রিন্ট**: স্বয়ংক্রিয় রশিদ জেনারেট হয় এবং প্রিন্ট করা যায়

## রিয়েল-টাইম সিঙ্ক

সিস্টেমটি রিয়েল-টাইমে দুই ব্যবহারকারীর মধ্যে ডাটা সিঙ্ক করে:

- **অটো সিঙ্ক**: প্রতি ৩০ সেকেন্ডে ডাটা সিঙ্ক হয়
- **তাৎক্ষণিক আপডেট**: একজন ব্যবহারকারীর পরিবর্তন অন্যজন দেখতে পায়
- **অ্যাক্টিভ ইউজার**: কোন কোন ব্যবহারকারী অনলাইনে আছে তা দেখা যায়
- **নোটিফিকেশন**: নতুন লেনদেনের জন্য নোটিফিকেশন পাওয়া যায়

## API ডকুমেন্টেশন

### গ্রাহক API
```
GET /api/customers.php?action=list          - গ্রাহকের তালিকা
GET /api/customers.php?action=search&term=কালাম - গ্রাহক খুঁজুন
POST /api/customers.php?action=add          - নতুন গ্রাহক যোগ করুন
POST /api/customers.php?action=update&id=1  - গ্রাহক আপডেট করুন
POST /api/customers.php?action=delete&id=1  - গ্রাহক ডিলিট করুন
```

### বাকি API
```
GET /api/debts.php?action=list              - বাকির তালিকা
GET /api/debts.php?action=customer&id=1     - নির্দিষ্ট গ্রাহকের বাকি
POST /api/debts.php?action=add              - নতুন বাকি যোগ করুন
POST /api/debts.php?action=update&id=1      - বাকি আপডেট করুন
POST /api/debts.php?action=delete&id=1      - বাকি ডিলিট করুন
```

### পেমেন্ট API
```
GET /api/payments.php?action=list           - পেমেন্টের তালিকা
GET /api/payments.php?action=debt&id=1      - নির্দিষ্ট বাকির পেমেন্ট
POST /api/payments.php?action=add           - নতুন পেমেন্ট যোগ করুন
POST /api/payments.php?action=delete&id=1   - পেমেন্ট ডিলিট করুন
```

### সিঙ্ক API
```
POST /api/sync.php                          - ডাটা সিঙ্ক করুন
```

## ডাটাবেস স্ট্রাকচার

### মূল টেবিল
- **users**: ব্যবহারকারী তথ্য
- **customers**: গ্রাহক তথ্য
- **debts**: বাকির হিসাব
- **payments**: পেমেন্টের হিসাব
- **notifications**: নোটিফিকেশন
- **activity_log**: অ্যাক্টিভিটি লগ
- **user_sessions**: ব্যবহারকারী সেশন

### সম্পর্ক
- একজন গ্রাহকের একাধিক বাকি থাকতে পারে
- একটি বাকির একাধিক পেমেন্ট থাকতে পারে
- প্রতিটি লেনদেন একজন ব্যবহারকারী দ্বারা তৈরি হয়

## নিরাপত্তা

### বাস্তবায়িত নিরাপত্তা ব্যবস্থা
- **SQL Injection Prevention**: Prepared statements ব্যবহার
- **XSS Prevention**: Input sanitization এবং output escaping
- **CSRF Protection**: CSRF token validation
- **Session Security**: Secure session management
- **Password Security**: bcrypt hashing
- **Rate Limiting**: API request rate limiting
- **Input Validation**: সব ইনপুট ভ্যালিডেশন

### সুপারিশ
- নিয়মিত পাসওয়ার্ড পরিবর্তন করুন
- শক্তিশালী পাসওয়ার্ড ব্যবহার করুন
- পাবলিক ওয়াইফাই এ লগইন এড়িয়ে চলুন
- নিয়মিত ডাটাবেস ব্যাকআপ নিন

## ট্রাবলশুটিং

### সাধারণ সমস্যা

**ডাটাবেস কানেকশন ত্রুটি**
```
Error: SQLSTATE[HY000] [2002] Connection refused
সমাধান: config/database.php ফাইলে ডাটাবেস ক্রেডেনশিয়াল চেক করুন
```

**ফাইল পারমিশন ত্রুটি**
```
Error: Permission denied
সমাধান: logs ফোল্ডারের পারমিশন 755 সেট করুন
```

**Google OAuth ত্রুটি**
```
Error: redirect_uri_mismatch
সমাধান: Google Console এ রিডাইরেক্ট URI চেক করুন
```

**সিঙ্ক কাজ করছে না**
```
সমাধান: ইন্টারনেট সংযোগ চেক করুন, ব্রাউজার কনসোল চেক করুন
```

### লগ ফাইল
- **Error Log**: `logs/php_errors.log`
- **Access Log**: ওয়েব সার্ভার অ্যাক্সেস লগ
- **Activity Log**: ডাটাবেসের activity_log টেবিল
